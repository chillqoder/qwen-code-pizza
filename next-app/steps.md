# Roadmap разработки: Pizza Delivery App

Это руководство предназначено для поэтапной реализации проекта. **Не пытайся сделать всё сразу.** Выполняй шаги последовательно. После каждого шага останавливайся, проверяй результат и убедись, что нет ошибок в терминале или консоли браузера.

Если на каком-то этапе возникает ошибка — исправляй её, прежде чем переходить к следующему шагу.

---

## 🛑 Правила работы
1.  **Один шаг за раз.** Выполняй только текущий этап.
2.  **Проверка.** После написания кода запусти проект (`npm run dev`) и проверь функционал.
3.  **Спека.** Сверяйся с файлом `SPECIFICATION.md` для деталей реализации.
4.  **Стек.** Используй только технологии, указанные в спеке (Next.js, Supabase, Shadcn, Zustand, Framer Motion).

---

## Этап 1: Инициализация проекта и зависимости
**Цель:** Подготовить чистую среду разработки и установить все необходимые библиотеки.

**Задачи:**
1.  Создай новый Next.js проект (App Router, TypeScript, Tailwind CSS).
2.  Инициализируй **Shadcn/UI**.
3.  Установи дополнительные зависимости:
    *   `@supabase/supabase-js`
    *   `zustand`
    *   `framer-motion`
    *   `lucide-react`
    *   `react-hook-form`
    *   `zod`
    *   `@hookform/resolvers`
    *   `swiper` (или используй кастомный слайдер на Framer Motion, если так проще)
4.  Настрой алиасы путей в `tsconfig.json` (например, `@/components`, `@/lib`).

**✅ Проверка:**
- [ ] Проект запускается без ошибок (`npm run dev`).
- [ ] В терминале нет красных сообщений при сборке.
- [ ] В проекте появилась папка `components/ui` (от Shadcn).

---

## Этап 2: Настройка Supabase и Типов
**Цель:** Подготовить базу данных и типизацию для TypeScript.

**Задачи:**
1.  Создай файл `.env.local` и добавь туда `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
2.  Создай файл SQL миграции (или выполни в редакторе Supabase) для создания таблиц: `categories`, `products`, `orders` (согласно `SPECIFICATION.md`).
3.  Добавь немного тестовых данных (seed data) в таблицы через SQL Editor, чтобы было что отображать (минимум 3 категории, по 5 товаров в каждой).
4.  Создай файл `types/database.ts` и опиши интерфейсы для `Product`, `Category`, `Order`.
5.  Создай клиент Supabase в `lib/supabase.ts`.

**✅ Проверка:**
- [ ] В терминале нет ошибок подключения к env переменным.
- [ ] Ты можешь сделать простой `console.log` данных из Supabase на главной странице (test fetch).
- [ ] TypeScript видит типы данных (нет ошибок `any`).

---

## Этап 3: Глобальное состояние (Корзина)
**Цель:** Создать хранилище для корзины перед началом верстки UI.

**Задачи:**
1.  Создай стор Zustand в `store/useCartStore.ts`.
2.  Реализуй методы: `addItem`, `removeItem`, `updateQuantity`, `clearCart`, `total`.
3.  Добавь persist middleware для сохранения корзины в `localStorage` (чтобы не терялась при перезагрузке).
4.  Создай тип `CartItem`.

**✅ Проверка:**
- [ ] Код стора компилируется без ошибок TypeScript.
- [ ] В консоли браузера (React DevTools или просто логика) можно добавить товар и увидеть, что состояние меняется.
- [ ] После перезагрузки страницы товары остаются в корзине (проверка localStorage).

---

## Этап 4: Layout и Header (Шапка)
**Цель:** Создать каркас приложения и закрепленную навигацию.

**Задачи:**
1.  Создай компонент `Header` в `components/layout/Header.tsx`.
2.  Реализуй `position: sticky`, `top: 0`, `z-index: 50`, blur-эффект фона.
3.  Добавь Логотип (текст или SVG).
4.  Добавь иконку корзины (`Lucide ShoppingCart`) с бейджем количества товаров (подключи Zustand).
5.  Добавь горизонтальный список категорий (табы). Пока они могут быть статичными или fetched из БД.
6.  Реализуй плавный скролл к секциям при клике на таб (якорные ссылки).

**✅ Проверка:**
- [ ] Шапка прилипает к верху при скролле.
- [ ] Фон шапки полупрозрачный.
- [ ] При клике на категорию страница скроллится к нужному месту.
- [ ] Иконка корзины отображает актуальное число товаров (из Zustand).

---

## Этап 5: Hero Slider (Баннер)
**Цель:** Реализовать главный визуальный элемент.

**Задачи:**
1.  Создай компонент `HeroSlider` в `components/home/HeroSlider.tsx`.
2.  Установи высоту контейнера `600px` (на мобильных адаптируй через `md:h-[600px] h-[400px]`).
3.  Реализуй слайдер (автоплей 5 секунд, переключение).
4.  Добавь анимацию появления текста/кнопки через `Framer Motion`.
5.  Используй заглушки изображений (например, Unsplash), если нет реальных фото.

**✅ Проверка:**
- [ ] Слайдер занимает правильную высоту.
- [ ] Слайды переключаются автоматически.
- [ ] На мобильных устройствах слайдер не ломает верстку.
- [ ] Нет ошибок в консоли, связанных с размерами изображений (Next/Image).

---

## Этап 6: Секции товаров и Карточки
**Цель:** Отобразить каталог продукции.

**Задачи:**
1.  Создай компонент `ProductCard` в `components/products/ProductCard.tsx`.
2.  Стилизуй карточку (фото, название, цена, кнопка).
3.  Создай компонент `CategorySection`, который принимает список товаров.
4.  Реализуй горизонтальный скролл (`overflow-x-auto`, `snap-x`) для списка карточек.
5.  На главной странице (`page.tsx`) сделай fetch категорий и товаров из Supabase и отрисуй секции.
6.  Настрой сетку: 4 карточки в ряд на десктопе, адаптив на мобильных.

**✅ Проверка:**
- [ ] Товары загружаются из базы данных.
- [ ] Горизонтальный скролл работает плавно.
- [ ] Карточки выглядят аккуратно, цены читаемы.
- [ ] Изображения оптимизированы (Next/Image).

---

## Этап 7: Модальное окно товара
**Цель:** Добавить возможность просмотра деталей товара.

**Задачи:**
1.  Используй компонент `Dialog` из Shadcn.
2.  Создай компонент `ProductModal`.
3.  При клике на `ProductCard` открывай модалку и передавай туда данные товара.
4.  Внутри модалки: большое фото, описание, состав, кнопка "Добавить в корзину".
5.  Подключи кнопку "Добавить" к `useCartStore`.
6.  Добавь анимацию появления (`Framer Motion`).

**✅ Проверка:**
- [ ] Модалка открывается по клику на товар.
- [ ] Данные в модалке соответствуют выбранному товару.
- [ ] При нажатии "Добавить" счетчик в хедере увеличивается.
- [ ] Модалка закрывается по клику вне области или на крестик.

---

## Этап 8: Выдвижная корзина (Cart Drawer)
**Цель:** Реализовать интерфейс управления заказом.

**Задачи:**
1.  Используй компонент `Sheet` (position: right) из Shadcn.
2.  Создай компонент `CartDrawer`.
3.  Подключи открытие к иконке в `Header`.
4.  Внутри: список товаров (маппинг из Zustand), возможность менять количество (+/-), кнопка удаления.
5.  Внизу (footer): общая сумма и кнопка "Оформить заказ".
6.  Список товаров должен скроллиться, если их много, футер закреплен.

**✅ Проверка:**
- [ ] Корзина выезжает справа.
- [ ] Товары отображаются корректно.
- [ ] Изменение количества обновляет общую сумму мгновенно.
- [ ] Кнопка "Оформить заказ" ведет на страницу `/checkout` (пока пустую).

---

## Этап 9: Страница оформления заказа (Checkout)
**Цель:** Сбор данных клиента и отправка заказа.

**Задачи:**
1.  Создай страницу `app/checkout/page.tsx`.
2.  Создай форму (React Hook Form + Zod схема валидации).
3.  Поля: Имя, Телефон, Адрес, Комментарий.
4.  Отобрази сводку заказа (список товаров и сумма) только для чтения.
5.  Напиши Server Action или API route для отправки данных в таблицу `orders` в Supabase.
6.  При успешном заказе: очисти корзину (Zustand) и перенаправь на страницу `/success`.

**✅ Проверка:**
- [ ] Валидация работает (нельзя отправить пустую форму).
- [ ] Заказ сохраняется в базу данных Supabase (проверь таблицу `orders`).
- [ ] Корзина очищается после заказа.
- [ ] Происходит редирект на страницу успеха.

---

## Этап 10: Финальная полировка и Анимации
**Цель:** Сделать проект "живым" и готовым к показу.

**Задачи:**
1.  Добавь `AnimatePresence` для плавного появления элементов при скролле.
2.  Добавь анимацию "подпрыгивания" иконки корзины при добавлении товара.
3.  Проверь мобильную версию (открой DevTools -> Mobile). Исправь отступы, шрифты.
4.  Добавь обработку ошибок (Toast уведомления при ошибке заказа или загрузки).
5.  Убедись, что все `alt` у картинок заполнены.
6.  Проверь скорость загрузки (Lighthouse).

**✅ Проверка:**
- [ ] Нет ошибок в консоли (Console Clean).
- [ ] На мобильном телефоне всё удобно нажимать пальцем.
- [ ] Анимации не тормозят (60fps).
- [ ] Проект выглядит целостно.
