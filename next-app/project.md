# Техническая спецификация проекта: Pizza Delivery App

## 1. Обзор проекта
Создание современного, быстрого и анимированного веб-приложения для доставки пиццы.
**Стек:** Next.js (App Router), Supabase, Tailwind CSS.
**Цель:** Single Page Application опыт с плавными переходами, выдвижной корзиной и удобной навигацией.

## 2. Технологический стек (Vibe Stack)
Используй следующие библиотеки для реализации. Не выдумывай свои, используй проверенные решения для ускорения разработки:

*   **Framework:** Next.js 14+ (App Router).
*   **Styling:** Tailwind CSS.
*   **UI Components:** Shadcn/UI (используй компоненты: Button, Card, Dialog, Sheet, Input, Label, Select, Toast).
*   **Animations:** Framer Motion (для всех анимаций: слайдер, появление модальных окон, выезд корзины).
*   **Icons:** Lucide React.
*   **State Management:** Zustand (для управления состоянием корзины).
*   **Forms:** React Hook Form + Zod (для валидации формы заказа).
*   **Backend/DB:** Supabase (PostgreSQL).
*   **Images:** Next/Image (оптимизация).

## 3. Структура Базы Данных (Supabase)
Создай следующие таблицы в Supabase. Типы данных должны соответствовать PostgreSQL.

### Таблица `categories`
*   `id` (uuid, primary key)
*   `name` (text) - например: "Пицца", "Напитки", "Десерты"
*   `image_url` (text) - иконка или фото категории
*   `order` (int) - для сортировки

### Таблица `products`
*   `id` (uuid, primary key)
*   `name` (text)
*   `description` (text)
*   `price` (int) - цена в копейках или целых числах
*   `image_url` (text)
*   `category_id` (uuid, foreign key -> categories.id)
*   `ingredients` (text[]) - массив ингредиентов

### Таблица `orders`
*   `id` (uuid, primary key)
*   `created_at` (timestamp)
*   `customer_name` (text)
*   `customer_phone` (text)
*   `customer_address` (text)
*   `total_amount` (int)
*   `status` (text) - default 'new'
*   `items` (jsonb) - сохраненный состав заказа (snapshot)

## 4. Архитектура и Компоненты

### 4.1. Глобальный Layout
*   **Header (Шапка):**
    *   `position: sticky`, `top: 0`, `z-index: 50`.
    *   Фон: полупрозрачный с blur-эффектом (glassmorphism).
    *   **Слева:** Логотип.
    *   **Центр:** Горизонтальный скролл-список табов (Категории). Табы должны быть закреплены внутри хедера. При клике — плавный скролл к соответствующей секции на странице (`scroll-behavior: smooth`).
    *   **Справа:** Иконка корзины (Lucide `ShoppingCart`) с бейджем количества товаров. При клике открывает Drawer (Sheet).

### 4.2. Главная страница (`/page.tsx`)
Страница состоит из следующих секций:

1.  **Hero Slider (Баннер):**
    *   Высота: `600px` (на мобильных адаптивно меньше, например `400px`).
    *   Библиотека: `Swiper.js` или кастомный слайдер на `Framer Motion`.
    *   Контент: Крупные промо-фото акций, заголовок, кнопка "Заказать".
    *   Автоплей: 5 секунд.

2.  **Секции товаров (Categories Sections):**
    *   Для каждой категории из БД рендерится отдельная секция с `id={category.id}`.
    *   Заголовок секции (например, "Пицца").
    *   **Сетка товаров:**
        *   Горизонтальный скролл контейнер (`overflow-x-auto`, `snap-x`).
        *   Карточки товаров выстраиваются в ряд.
        *   **Адаптивность:** На десктопе видно 4 карточки в ряд, на планшете 2-3, на мобильном 1.5 (чтобы было видно край следующей).
        *   Скроллбар скрыт или стилизован минималистично.

3.  **Карточка товара (Product Card):**
    *   Компонент `Card` из Shadcn.
    *   Изображение (aspect ratio 1/1 или 4/3).
    *   Название, цена, краткое описание.
    *   Кнопка "В корзину" (быстрое добавление) ИЛИ клик по всей карточке открывает Модалку. *Решение: Клик по карточке открывает Модалку, внутри модалки кнопка добавления.*

### 4.3. Модальное окно товара (Product Modal)
*   Компонент `Dialog` из Shadcn.
*   Открывается при клике на карточку товара.
*   **Контент:**
    *   Большое фото слева (или сверху на мобильном).
    *   Название, цена, полное описание, состав.
    *   Выбор размера/добавок (опционально, если есть в БД).
    *   Кнопка "Добавить в корзину" (большая, яркая).
    *   Анимация появления: `scale` + `fade` через Framer Motion.

### 4.4. Корзина (Cart Drawer)
*   Компонент `Sheet` (Side Sheet) из Shadcn, позиция `right`.
*   Открывается по клику на иконку в хедере.
*   **Внутри:**
    *   Заголовок "Ваш заказ".
    *   Список товаров (Scrollable area, `flex-1`, `overflow-y-auto`).
    *   Элемент списка: Фото (мини), Название, Цена, Контроллер количества (+ / - / Удалить).
    *   **Футер корзины (закреплен внизу):**
        *   Итоговая сумма (крупно).
        *   Кнопка "Оформить заказ" (ведет на `/checkout`).
*   Состояние корзины хранится в **Zustand store** + синхронизация с `localStorage`.

### 4.5. Страница оформления заказа (`/checkout/page.tsx`)
*   Отдельная страница.
*   **Слева:** Форма заказа (React Hook Form).
    *   Поля: Имя, Телефон, Адрес (textarea), Комментарий.
    *   Валидация через Zod (телефон обязателен, адрес обязателен).
*   **Справа:** Сводка заказа (как в корзине, но без возможности редактирования количества, только просмотр).
*   **Действие:** При сабмите формы:
    1.  Создается запись в таблице `orders` в Supabase.
    2.  Очищается корзина (Zustand).
    3.  Редирект на страницу "Спасибо" (`/success`).

## 5. Логика и State Management (Zustand)

Создай стор `useCartStore`:
*   `items`: Array<{ product_id, quantity, price, name, image }>.
*   `addItem(product)`: Добавляет товар. Если уже есть — увеличивает quantity.
*   `removeItem(id)`: Удаляет товар.
*   `updateQuantity(id, delta)`: Меняет количество.
*   `clearCart()`: Очищает всё.
*   `total`: Геттер, считающий сумму.

## 6. Требования к UI/UX (Vibe)
*   **Цветовая гамма:** Теплая, аппетитная (оранжевый, красный, белый фон, темный текст). Используй палитру Tailwind `orange-500` как акцент.
*   **Шрифты:** Современный гротеск (Inter или Manrope).
*   **Анимации:**
    *   При добавлении в корзину — иконка корзины в хедере должна "подпрыгивать".
    *   Плавное появление контента при скролле (Fade In Up).
    *   Ховер-эффекты на карточках (легкое увеличение scale).
